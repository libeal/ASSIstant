# 执行权限边界声明（生产加固）

本文档定义 Agent 各执行通道的权限边界，是 [`docs/未完成计划.md`](../docs/未完成计划.md) 阶段 1「非 root 执行与提权隔离」的落地说明。配套件：[`packaging/linux-agent-web.service`](linux-agent-web.service)。

## 运行身份模型

- **Agent 主进程以专用非 root 用户运行**（推荐 `linux-agent`），由 systemd 单元的 `User=`/`NoNewPrivileges=` 保证。
- 目标主机上的运维工程师通过 SSH 登录后，经 SSH 端口转发访问 Web 控制台的 `127.0.0.1:8765`；SSH 提供传输加密与身份。
- 当 Agent 以 root 运行时（不推荐，仅兼容旧部署），每一步命令经最小权限代理 `runuser`/`setpriv` 降权到 `execution.least_privilege_user`（默认 `nobody`），见 [`lib/executor.sh`](../lib/executor.sh) `linux_agent_prepare_execution_command`。

## 各执行通道的边界

| 通道 | 执行方式 | 权限边界 | 密钥可见性 |
|---|---|---|---|
| Terminal（shell） | `bash -lc` | 非 root 用户；root 时降权到 `nobody` | 已剥离 API key（`env -u`） |
| Skill 脚本 | `bash <script> <json>` | 同上 | 已剥离 API key |
| MCP 工具 | `python3 mcp_client.py call-tool …` | 同上；stdio server 环境额外剥离密钥 | 已剥离 API key（executor + mcp_client 双层） |
| 远程脚本 | 下载后审阅再 `bash <path>` | 同上；HTTPS 下载 + 大小/文本校验 | 已剥离 API key |
| 文件编辑 | `controlled-tools` skill | 经 file_vault 分类 + policy 审查 | 已剥离 API key |
| AI 调用 | `curl`（仅 `lib/ai.sh`） | 主进程内发起，不经执行通道 | **唯一**允许持有 API key 的路径 |

**密钥最小作用域**：API key 仅注入 AI 调用路径。所有执行通道的子进程都通过 `env -u LINUX_AGENT_API_KEY …`（executor）与 `os.environ` 剥离（mcp_client）确保看不到密钥。Web 适配层 `agent_subprocess_env` 只对 `work/run`、`edit/plan` 注入密钥。

## systemd 沙箱要点

`packaging/linux-agent-web.service` 施加：

- `NoNewPrivileges`、`ProtectSystem=strict`、`ProtectHome`、`PrivateTmp`；仅将 `/opt/linux-agent/config`、`/opt/linux-agent/logs` 和 `/opt/linux-agent/tmp` 作为可写运行时目录；
- `SystemCallFilter=@system-service` 去除 `@privileged/@resources/@mount/@swap/@reboot/@raw-io`；
- cgroup：`CPUQuota=200%`、`MemoryMax=1G`、`TasksMax=256`；
- 网络：Web 仅监听回环；出口可用 `IPAddressAllow/Deny` 收敛到 Provider 网段。

## 提权 helper（扩展点）

需要提权的能力（如启用 auditd observer）**不应**让整个 Agent 进程 root 化。目标形态是一个独立的 privileged helper：

- 由 root 或带 `CAP_AUDIT_CONTROL` 的最小单元承载；
- 通过受控接口（unix socket / 明确命令白名单）接受 Agent 的降权请求；
- 审计记录请求用户、执行用户与提权路径（见 [`lib/audit.sh`](../lib/audit.sh) 的 `execution_proxy` 字段）。

当前版本提供边界声明与 systemd 约束；privileged helper 的具体实现属后续里程碑。
