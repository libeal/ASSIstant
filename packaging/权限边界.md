# 执行权限边界声明（生产加固）

本文档定义 Agent 各执行通道的权限边界，是 [`docs/未完成计划.md`](../docs/未完成计划.md) 阶段 1「非 root 执行与提权隔离」的落地说明。配套件：[`packaging/linux-agent-web.service`](linux-agent-web.service)。

## 运行身份模型

- **Agent 主进程以专用非 root 用户运行**（推荐 `linux-agent`），由 systemd 单元的 `User=`/`NoNewPrivileges=` 保证。
- 目标主机上的运维工程师通过 SSH 登录后，经 SSH 端口转发访问 Web 控制台的 `127.0.0.1:8765`；SSH 提供传输加密与身份。
- 当 Agent 以 root 运行时（不推荐，仅兼容旧部署），每一步命令经最小权限代理 `runuser`/`setpriv` 降权到 `execution.least_privilege_user`（默认 `nobody`），见 [`lib/executor.sh`](../lib/executor.sh) `linux_agent_prepare_execution_command`。

## 各执行通道的边界

| 通道 | 执行方式 | 权限边界 | 密钥可见性 |
|---|---|---|---|
| Terminal（shell） | `bash -lc` | 非 root 用户；root 时降权到 `nobody` | 显式白名单环境，不继承父进程凭据 |
| Skill 脚本 | `bash <script> <json>` | 同上 | 显式白名单环境，不继承父进程凭据 |
| MCP 工具 | `python3 mcp_client.py call-tool …` | 同上；stdio server 使用独立白名单环境 | manifest 仅能显式注入自身配置，不能覆盖 Agent 密钥 |
| 远程脚本 | 下载后审阅再 `bash <path>` | 同上；HTTPS 下载 + 大小/文本校验 | 显式白名单环境 |
| 文件编辑 | `controlled-tools` skill | 经 file_vault 分类 + policy 审查 | 显式白名单环境 |
| AI 调用 | `curl`（仅 `lib/ai.sh`） | 主进程内发起，不经执行通道 | **唯一**允许持有 API key 的路径 |

**密钥最小作用域**：API key 仅注入 AI 调用路径。Web、executor 和 MCP 都从空环境按白名单构造子进程环境，不再复制父进程环境；`work/run`、`edit/plan` 只把 key 交给 Bash 编排进程，实际 Terminal/Skill/MCP 步骤会再次清空环境。

## 生产目录与版本切换

`linux-agent-install.sh` 默认把只读代码安装到 `/opt/linux-agent/releases/<version>/`，并用原子更新的 `current` 符号链接选择当前版本。配置、日志和临时状态位于 `/opt/linux-agent/data/{config,logs,tmp}/`；每个版本内的同名目录都是指向持久数据的符号链接，因此升级和回滚不会覆盖运行数据。

`upgrade` 在切换后重启服务并轮询 `/api/health`，健康检查失败时自动切回旧版本。`rollback` 使用 `releases/.history` 选择上一个仍存在的版本。默认只保留最近两个版本，可通过 `--keep` 调整。卸载默认保留 `data/`，只有显式 `uninstall --purge-data` 才删除持久数据。

## systemd 沙箱要点

`packaging/linux-agent-web.service` 施加：

- `NoNewPrivileges`、`ProtectSystem=strict`、`ProtectHome`、`PrivateTmp`；仅将真实持久目录 `/opt/linux-agent/data` 作为可写运行时目录；
- `SystemCallFilter=@system-service` 去除 `@privileged/@resources/@mount/@swap/@reboot/@raw-io`；
- cgroup：`CPUQuota=200%`、`MemoryMax=1G`、`TasksMax=256`；
- 网络：Web 仅监听回环；首次 systemd 安装必须通过一个或多个 `--provider-cidr` 启用 `IPAddressDeny=any` + 显式放行，或由运维显式声明 `--allow-unrestricted-provider-egress`。

## auditd 提权 helper

启用 auditd observer 时不让整个 Agent 进程 root 化，而由独立 helper 承担唯一的提权边界：

- `linux-agent-observer-helper.service` 仅保留 audit 所需 capability，并由 systemd socket 激活；
- socket 为 `root:<service-group>`、`0660`，helper 再通过 `SO_PEERCRED` 核对请求进程；
- 协议只允许 audit 状态、规则列表、按 Agent key 查询，以及对固定 syscall allowlist 增删当前请求用户的规则；不接受命令文本、路径或任意 auditctl 参数；
- helper 使用 root 所有、不可被组/其他用户写入的绝对工具路径，限制请求/输出大小和执行时间，并为每次请求写 journal 事件；
- socket 存在但 helper 调用失败时 observer fail-closed，不回退到 sudo。显式设置 `observer.privilege=none` 会同时禁用 helper 与 sudo。

实现位于 [`lib/observer_helper.py`](../lib/observer_helper.py)，部署单元为 [`linux-agent-observer-helper.service`](linux-agent-observer-helper.service) 与 [`linux-agent-observer-helper.socket`](linux-agent-observer-helper.socket)。helper 只服务 auditd observer，不提供通用 root 命令能力。

安装器事务化管理 `linux-agent-web.service.d/10-provider-egress.conf`，升级失败时与 unit 一并恢复。模板 [`packaging/dropins/10-provider-egress.conf.example`](dropins/10-provider-egress.conf.example) 仅供未使用安装器的手工部署参考。策略默认 deny-any + localhost，由运维显式放行主/备用 Provider CIDR；没有可维护 CIDR 时必须显式选择无限制出口并接受安装器警告。
